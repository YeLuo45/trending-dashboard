# GitHub Trending 仪表盘 UI 重设计 & 移动端适配设计说明书 (UI Redesign Spec)

## 0. 设计定位与参数 (Design Read & Config)

* **Design Read**: B2B 与极客开发者取向的 GitHub 趋势深度分析数据仪表盘。采用 **Minimalist Premium Tech (现代极简高级科技风格)** 视觉语言，构建如 Vercel 或 Linear 般极其精细、高透光、功能主导的产品看板。
* **三大基础控制参数 (Three Dials)**:
  * `DESIGN_VARIANCE: 6` — 通过非对称的数据统计标签、偏置排版打破常规单调的对称网格卡片布局。
  * `MOTION_INTENSITY: 6` — 全局采用物理弹簧 (Spring) 曲线。组件渐入带有微弱的 Y 轴位移和级联延迟（Stagger），卡片及核心操作按钮支持高响应性的触觉回弹。
  * `VISUAL_DENSITY: 5` — 确保精细、紧凑的同时，让文字和数据获得呼吸感。摒弃多层嵌套边框，将元素合并在通透的毛玻璃底板（Glassmorphism Backdrop）中。

## 1. 材质与配色规范 (Material & Color Calibration)

### 1.A 色彩一致性锁定 (Color Consistency Lock)
为确保全页面色彩系统的极高纯净度与严谨感，避免散乱的色彩分布，全站锁定如下配色方案：
* **暗色模式背景 (Dark Theme Base)**: `#09090b` (Zinc-950) 作为全局深邃墨色底色。绝对禁止纯黑 `#000000` 或传统的 GitHub 蓝灰 `#0d1117`。
* **高阶极简卡片 (Transparent Panel)**:
  * 卡片背景: `rgba(18, 18, 22, 0.45)` (极暗锌灰微透)
  * 卡片边框: 极精细单色高光边缘 `border-white/5` 或 `border-zinc-800/60` (不使用突兀、厚重的灰色粗线)。
  * 毛玻璃模糊: `backdrop-blur-xl`。
  * 卡片内发光: `box-shadow: inset 0 1px 0 rgba(255, 255, 255, 0.03)` (模拟玻璃边缘折射光，避免扁平单调)。
* **单主轴点缀色 (Accent Lock)**: 全局换用 **科技靛蓝 (Digital Indigo, #6366f1 / Tailwind indigo-500)**。
  * 已选状态、活跃标签滑块、推荐度标签、星标按钮、主要动作 CTA 统一使用该靛蓝作为唯一亮色指引。
  * 彻底清理、弃用以往混乱突兀的 AI Lila 紫色。
* **亮色模式适配 (Light Theme Override)**:
  * 亮色模式下，全局背景切换为纯净的纸张灰色 `#f9f9fb`，卡片采用带有极弱底阴影的白色背景 `rgba(255, 255, 255, 0.8)` + `border-zinc-200/50`。点缀色换为中强度的靛蓝色 `#4f46e5`。

### 1.B 形状圆角锁定 (Shape Consistency Lock)
* 核心卡片容器: `rounded-2xl` (16px)。
* 按钮、切换组件、搜索框等交互单元: `rounded-xl` (12px)。
* 头像、状态圆点、全选中复选框、趋势升降徽章: 全圆胶囊 `rounded-full`。
* 严禁混合使用直角与大圆角，所有交互区域维持统一度规。

## 2. 精密排版体系 (Precision Typography)

* **展示标题 (Header Displays)**:
  * 字体家族: 现代、收紧的 Sans-serif Display（极简科技）。
  * 标题字距: `-0.03em` tracking (收紧字间距，展现重磅和高张力)。
  * 主页大标题 "GitHub Trending" 样式调优为粗黑 sentence-case，副标题去除多余修饰语，强调 “精密追踪技术风向标”。
* **等宽数字指标 (Tabular & Monospace Metrics)**:
  * **所有涉及数值** 的显示区（Rank 排名、Stars 总星数、Forks 分支数、Growth 日/周/月增长数）一律使用代码等宽字体 `font-mono` (例如 `JetBrains Mono` 或 `Geist Mono`) 渲染。
  * 在进行多卡片横向与纵向扫描对比时，等宽数字可提供卓越的对齐易读性。
* **正文描述 (Description Copies)**:
  * 限制单行最长可读字符数在约 65 个半角字符内（`max-w-[65ch]`）。
  * 行高设定为 `leading-relaxed` (1.625)，字号 `text-[13.5px]` 到 `text-[14px]`，使用 Muted 锌灰色（`text-zinc-400`）。
  * 严禁使用任何 `—` (Em-dash) 作为文字修饰，所有连接或隔离全部换用微弱竖线 `|`、换行或普通短横线 `-`。

## 3. 移动端 UI 原生化重设计 (Mobile-First Architecture)

原系统直接隐藏大批桌面控制按钮（`hide-mobile`），并把功能揉杂在简陋的 `MobileDrawerNav`（☰ 按钮）中。我们将按纯原生体验进行适配重构：

### 3.A 悬浮底座导航栏 (Mobile Navigation Dock)
* **大拇指交互热区 (Touch Targets)**:
  * 在屏幕宽度 `< 768px` 时，固定在屏幕底端放置一个精致的悬浮式半透毛玻璃导航栏底座（Floating Bottom Dock）。
  * 底座高度锁定在 `64px`，离屏幕底端安全区预留 `12px` (避免与 iOS 底部小白条冲突)，两边留白以展示悬浮科技质感。
  * 提供四大核心大触控按钮（触控热区均不低于 `48px * 48px`）：
    1. **「趋势看板 (Feed)」** (默认激活，对应每周/最热/每日等主视图)
    2. **「高级筛选 (Filters)」** (点击直接从底部平滑推出半屏 bottom sheet)
    3. **「我的收藏 (Favorites)」** (点击划入收藏抽屉，可在此分享和生成海报)
    4. **「更多功能 (More)」** (点击划入包含 Fork 历史、话题追踪、导出数据、通知、设置的系统汇总抽屉)

### 3.B 移动端底部半屏抽屉 (Bottom Sheets)
* **交互形态**: 所有社交、工具面板在移动端一律不再强制全屏弹窗。而是平滑地从屏幕底部滑入一个 **底部弹性卡抽屉 (Bottom Sheet)**。
* **操作动效**: 滑动阻尼效果使用 `motion/react` 弹簧规格（`stiffness: 200, damping: 24`），配合大拇指下拉手势可轻松收起，极为自然。
* **卡片紧凑排布**: 在手机窄屏上，去掉 ProjectCard 中不必要的繁杂标签，将 Rank 标志采用小胶囊左上定位，核心的 Stars、Forks 和 Growth 采用简洁并列底横栏布局。

## 4. 关键页面组件微观设计升级 (Components Polish)

### 4.A Header 头部组件
* **桌面端**: 清理原版塞满 8 个并排图标按钮的臃肿乱象。重新设计为：
  - 左侧：极简 Logo 与 Sentence-case 标题、最后更新时间（最后更新时间不占高，使用小字精巧呈现在左上角或标题下方）。
  - 右侧：常驻【全局状态：登录 GitHub / 未登录 ⚙️】、一键【生成分享海报】以及【全局通知 🔔】。
  - 核心的高级辅助面板（收藏、追踪、报告、推荐、导出、历史等）在桌面端折叠到一个更具仪式感的 **「极客智能分析套件」下拉聚合菜单** 中。
* **移动端**: 菜单栏彻底收归至 Mobile Dock 集中托管。

### 4.B ProjectCard 趋势项目卡片
* **底板材质**: 卡片背景采用透气半透暗锌灰。去掉生硬黑边框。
* **交互反馈**: 
  - `:hover` 激活时，卡片呈现极为精致的轻微向内发光 (`border-indigo-500/25` / `shadow-[0_0_24px_rgba(99,102,241,0.08)]`)，整体略微抬升 `-1px`。
  - `:active` (触摸或点击按压) 时，弹簧缩放至 `scale-[0.98]`。
* **按钮精细化**: 
  - “Fork” 按钮和 “评论” 按钮均采用高度契合卡片格调的极微边框轻量级设计，并保证其背景与文字对比度不低于 4.5:1，以满足严格的 a11y 辅助功能测试。

### 4.C 状态过渡 (States & Micro-interactions)
* **等宽微动**: TabButton 滑动切换时，靛蓝色的指示条在 Tab 间以物理滑行过渡（`layoutId` 指导）。
* **通知气泡**: 带有新项目提示的铃铛气泡轻柔脉动（Pulse），引导而不吵闹。
* **骨架屏**: 加载时的 Skeleton 结构长宽厚度与 ProjectCard 一一等比例对应。

## 5. 严格规范与质量自审 (Strategic Omissions & Rules)

* **[x] 绝对零 Em-dash 准则**: 在标题、副标题、文案、属性分隔区不出现任何 `—`。一律使用标准逗号、句号、英文短横线 `-`。
* **[x] 无过度渐变字**: 严禁在全站标题中使用大面积、多色彩的炫目渐变。全站只采用单色极高对比度排版，仅在最核心的数据标签里使用柔和的点缀。
* **[x] 真实而自然的假数据**: 避免诸如 `99.9%`、`100` 这种机械假数据，全系统统一提取真实历史沉淀或格式化后的原生数据格式。
* **[x] 无多余的卡片套卡片**: 杜绝在 Tab 卡片内又套大面积边框阴影小卡片的臃肿排版，靠精美的间距与极细 Hailline 排列展现专业度。
